Before We Start

ilone is a one paged html template powerd by ajax. Because page transitions are done by loading separate html files into the main index.html with ajax, it can not run with "file:" protocol. It requires a local (localhost) or remote hosted server to work.


A) HTML Structure - top

The structure of this template is very intuitive and easy to understand. Let's dive in to the main structure itself.

*Sections: Each section represents a page, so when you declare a section with proper id and a template class, Javascript automatically applies the neccesary code to the section and enables the functionality automatically. There is a structure inside sections that must be fallowed in order to make it work.


Section structure

First you have to have a section with proper id and template class.
In this case we have applied the "works-template" which will take care of the Css Style and of course the proper JavaScript will be applied to the section enabling Works section functionality.

There is a very important part in sections that must exist in every section in order for JavaScript to work. It is a "DIV" element with class "section-holder".
"section-holder", is the element that is animated in sections like "Works" ("works-template") and "Blog" ("blog-template") when there is a page swith. So, This is the main structure for sections that make the pageswitch possible, you may have anything inside "section-holder" element and it will still be animate if that particular section supports page swith.
If the page switch is supported, there is an anchor element with a special class that initiates the pageswitch. Check each section explanation in order to find the special class for those.

Section id:
Another importat part in sections is the ID. The main naviagtion relies on the id of each section plus if the deeplinking is enabled section id will be used as a pointer to the section where the interactions occur.
Deeplink functionality will be explained further in JavaScript section.

B) Box Loader - top

Box Loader is a custom module that makes it possible to load html data from a file into the main page, like the "Team/Member Viewer" in the preview. Below is the explanation of its functionality and structure.

Lets take a look at the classes and their purpose.
box-loader: This element is the main container for the Box Loader module. Without it the module will not be initiated thus will not work. 

			
  • 
				
			
    •
    bl-activeate-on-child-link: This class is used on the parent or grand-parent of the main anchor element that initiates the page load. It makes sure that the active class will be applied to itself instead of the anchor. So if you don't need to have separate containers/wrapper for each of your anchor elements you don't have to use this class. If no "bl-activeate-on-child-link" is found on the active anchor's parent or grand-parent then the active class will be applied to anchor itself. 

    			
				Link Label
    box-loader-link: Any anchor, within the main Box Loader ("box-loader") container, that has a class of "box-loader-link" will initiate page/data load.

    box-loader-link href="": The structure of the href is quit intuitive. It is split into two parts, the url of the html file ("sample-html-files/sample.html") followed by a hash tag (#) and the id (sample-element-id) of the element which resides in the loaded html file that will be loaded into the "box-loader-data".

    box-loader-activate: This class can be applied on one of "box-loader-link" anchor elements. It will force the Box Loader module to load the data according to href value of that anchor when the main page is loaded.
    box-loader-data: This element is the container for newly loaded data. It will hold the loaded data/element when data is requested by a click on any of the "box-loader-link" anchor elements.

    Let's summarize, A Box Loader module will be applied to an element with class "box-loader" which contains the "box-loader-link", "box-loader-data". In an event of data request when user clicks on any of "box-loader-link" anchors, "box-loader-data" will animate out, then Box Loader module will load the data, append the newly loaded data into "box-loader-data" element, then animate "box-loader-data" back in applying the active class on the anchor element that was clicked or on a parent of that "box-loader-link" if "bl-activeate-on-child-link" class exists on any of its parents.


    C) Media Box / LightBox - top

    Media Box is a custom module which loads images, youtube, vimeo videos and websites in a lightbox manner. Greate thing about this module is that it recognizies the media type itself. If it is an image it will load the image, if a youtube or vimeo video it will load the video player. Lets see how it works.

    Loading Single Media - Image
    This will make the Media Box to load the image from the "big-image-url/image.jpg" url.

    Loading Single Media - Youtube
    Media Box will automatically recognize that this is a Youtube video and will load Youtube player with the specified url. You may use either the long or short url.

    Loading Single Media - Vimeo
    Media Box will automatically recognize Vimeo video and load Vimeo player.

    Loading Single Media - iframe
    Media Box also can load websites into an iframe. Note that some websites have some restrictions that will prevent the website to be loaded Exp: http://google.co.uk

    title: Title attribute of anchor element will be used to show a title under the loaded media if title value is set.

    data-dimension: data-dimension attribute is used to set the dimensions of the media. The structure is WIDTHxHEIGHT. So, let's say you need a Youtube video with 400px width and 300px height, you need to set it like so: 
    			Youtube Video



    Making a gallery
    In order to make a gallery you have to create an element with class "media-gallery" and put all the links into it. Media Box will automatically detect that these links are in a gallery and will enable next and previous navigation.
